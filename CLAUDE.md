# AI Scenarios Platform — CLAUDE.md

## Project Overview

**AI Scenarios** (aiscenarios.org) is a static site that tracks and evaluates transformative AI scenarios, inspired by the [Climate Action Tracker](https://climateactiontracker.org/) and modelled on [METR](https://metr.org/)'s clean, research-credible aesthetic.

It answers one question: **How prepared are we for transformative AI?**

The site ingests structured CSV data (scenarios, evaluations, policies), compiles it into chart-ready JSON, and renders an interactive homepage with a signature scatter plot showing scenario risk, impact, and preparedness.

### Design References
- **Climate Action Tracker**: The "thermometer" headline stat, country-level drill-downs, bold red/green rating system
- **METR**: Clean, serious, research-org aesthetic. White background, restrained typography, credible but accessible

```
ai-scenarios-platform/
├── CLAUDE.md              # This file — project spec

├── data/
│   ├── scenarios.csv      # 10 transformative AI scenarios
│   ├── evaluations.csv    # Scored evaluations (may have multiple per scenario over time)
│   └── policies.csv       # Policy proposals with citations
├── scripts/
│   └── compile.js         # CSV → JSON compiler (run via `npm run compile`)
├── site/
│   ├── index.html         # Homepage (static, references compiled data)
│   ├── chart.js           # Interactive scatter plot (vanilla JS + D3 or Chart.js)
│   ├── style.css          # Styles
│   └── data.json          # ← Generated by compile.js
└── docs/                  # Optional: methodology, about page
```

**Hosting**: GitHub Pages from the `site/` directory.

## Data Model

### scenarios.csv

| Column | Type | Description |
|--------|------|-------------|
| `id` | string | Short kebab-case ID, e.g. `tbi-whirlwind` |
| `title` | string | Full scenario title |
| `authors` | string | Author names (comma-separated) |
| `institution` | string | Publishing org |
| `url` | string | Link to original source |
| `year` | int | Publication year |
| `summary` | string | 1-2 sentence summary |
| `social` | string | STEEP: Social implications (1-2 sentences) |
| `technological` | string | STEEP: Technological assumptions |
| `economic` | string | STEEP: Economic impacts |
| `environmental` | string | STEEP: Environmental dimensions |
| `political` | string | STEEP: Political/governance implications |
| `tag_mass_unemployment` | boolean | TRUE/FALSE |
| `tag_fiscal_crisis` | boolean | |
| `tag_inequality` | boolean | |
| `tag_ai_concentration` | boolean | |
| `tag_skills_obsolescence` | boolean | |
| `tag_public_service_disruption` | boolean | |
| `tag_democratic_erosion` | boolean | |
| `tag_existential_risk` | boolean | |

The boolean tags represent **policy challenges** — each maps to a category of policy response.

### evaluations.csv

| Column | Type | Description |
|--------|------|-------------|
| `date` | ISO date | When evaluation was made |
| `scenario_id` | string | References `scenarios.id` |
| `likelihood` | int 0–100 | How likely is this scenario? |
| `impact` | int -100 to 100 | Unmitigated impact (negative = harmful) |
| `preparedness` | int 0–200 | How prepared are current policies? (0 = not at all, 200 = fully) |
| `likelihood_rationale` | string | Text justification |
| `impact_rationale` | string | Text justification |
| `preparedness_rationale` | string | Text justification |

**Important**: There may be multiple evaluations per scenario over time. The site should use the **latest** evaluation (by date) for each scenario.

### policies.csv

| Column | Type | Description |
|--------|------|-------------|
| `id` | string | Kebab-case policy ID |
| `name` | string | Policy name |
| `challenge_tag` | string | Which policy challenge this addresses (matches scenario boolean tags) |
| `description` | string | 2-3 sentence description |
| `citation_1_text` | string | Citation description |
| `citation_1_url` | string | URL |
| `citation_2_text` | string | Citation description |
| `citation_2_url` | string | URL |
| `citation_3_text` | string | Citation description |
| `citation_3_url` | string | URL |

## Compile Script (`npm run compile`)

The `scripts/compile.js` script should:

1. Read all three CSVs from `data/`
2. For evaluations, **deduplicate by scenario_id**, keeping only the row with the latest `date`
3. Join scenarios with their latest evaluation
4. Compute aggregate stats:
   - **Overall preparedness score**: Weighted average of all scenario preparedness scores (weighted by likelihood)
   - **Total scenarios**: Count
   - **Total policies**: Count
5. Output `site/data.json` with structure:

```json
{
  "meta": {
    "generated": "2026-03-02T...",
    "overallPreparedness": 21,
    "overallRating": "Low",
    "scenarioCount": 10,
    "policyCount": 15
  },
  "scenarios": [
    {
      "id": "...",
      "title": "...",
      "authors": "...",
      "institution": "...",
      "url": "...",
      "summary": "...",
      "steep": { "social": "...", "technological": "...", ... },
      "tags": ["mass_unemployment", "fiscal_crisis", ...],
      "evaluation": {
        "date": "...",
        "likelihood": 75,
        "impact": -80,
        "preparedness": 30,
        "rationale": { "likelihood": "...", "impact": "...", "preparedness": "..." }
      }
    }
  ],
  "policies": [
    {
      "id": "...",
      "name": "...",
      "challengeTag": "...",
      "description": "...",
      "citations": [
        { "text": "...", "url": "..." }
      ]
    }
  ]
}
```

Rating thresholds for `overallRating`:
- 0–40: "Critically Unprepared" (red)
- 41–80: "Low" (orange-red)
- 81–120: "Moderate" (yellow)
- 121–160: "Good" (light green)
- 161–200: "Well Prepared" (green)

## Homepage Design

### Layout (top to bottom)

#### 1. Hero Section
Split layout:
- **Left**: Headline stat
  ```
  Our current economic preparedness
  for transformative AI is:
  
  LOW (21%)          ← Big, bold, colored by rating
  
  [View Repo]  [Read Paper]
  ```
- **Right**: The interactive scatter chart (see below)

#### 2. Chart Controls
Three toggle buttons below the chart: **Net Risk** | **Impact** | **Preparedness**

These change what the chart displays:
- **Net Risk** (default): X = likelihood, Y = impact × (1 - preparedness/200). Higher = worse. The "danger zone" (top-right quadrant) is shaded red.
- **Impact**: X = likelihood, Y = raw unmitigated impact
- **Preparedness**: X = likelihood, Y = preparedness score

#### 3. How It Works
Brief explainer text:
> "The future is uncertain — yet we must prepare. That's why we've evaluated **{scenarioCount} scenarios** and tested the effectiveness of **{policyCount} policy proposals** to gauge our AI readiness."

The bold numbers should be dynamically pulled from data.json.

#### 4. Scenario Table
A summary table showing each scenario with:
- Scenario name
- Top 3 relevant policies
- Readiness rating (e.g., "2/5" as stars or a small bar)

### Interactive Chart Spec

- **Type**: Scatter plot
- **Library**: D3.js (preferred for control) or Chart.js
- **Points**: Each scenario is a dot, labeled with a short name (truncated if needed)
- **Hover**: Tooltip showing scenario title, likelihood, impact, preparedness, and 1-line summary
- **Click**: Could expand to show full STEEP analysis and relevant policies (stretch goal)
- **Danger zone**: In "Net Risk" mode, the top-right quadrant (high likelihood + high negative impact) should have a red/pink gradient background, matching the sketch
- **Responsive**: Must work on mobile

### Visual Design

- **Aesthetic**: Clean, serious, credible. Think research org, not startup.
- **Colors**: 
  - Primary: Deep navy (#1a1a2e) or dark charcoal
  - Accent: Alert red (#e74c3c) for danger/low preparedness
  - Secondary: Muted blues, warm grays
  - Success: Forest green for high preparedness
- **Typography**: 
  - Headings: A strong serif (e.g., "Instrument Serif" or "Playfair Display" from Google Fonts)
  - Body: Clean sans-serif (e.g., "Source Sans 3")
- **Layout**: Max-width container (~1100px), generous whitespace

## Seed Data Notes

The `data/` CSVs should be pre-populated with ~10 real transformative AI scenarios. Good candidates include:

1. **Situational Awareness** — Leopold Aschenbrenner's essay on superintelligence trajectory
2. **The Whirlwind** — Tony Blair Institute scenario on rapid AI transformation
3. **Machines of Loving Grace** — Dario Amodei's optimistic AI future
4. **The Compendium** — Epoch AI comprehensive economic analysis
5. **AI 2027** — AI forecasting scenario from Amodei/others on near-term trajectory
6. **White Collar Wipeout** — Mass professional job displacement scenario
7. **The Precipice** — Toby Ord's existential risk framing applied to AI
8. **Governing AI** — Mustafa Suleyman's governance framework scenario
9. **RAND AI & National Security** — Defense/security focused transformation
10. **IMF AI & the Economy** — Fiscal and macroeconomic disruption scenario

Evaluations and policies should be plausible seed data. The evaluations should reflect genuine expert uncertainty — most scenarios should have low preparedness scores.

## Key Principles

1. **Data-driven**: Everything on the site is derived from the CSVs. Edit the data, recompile, redeploy.
2. **Transparent**: All data and methodology should be inspectable. Open-source repo.
3. **Credible**: The aesthetic and tone should feel like a serious research project, not a blog.
4. **Simple**: No build tools beyond Node. No frameworks. Just CSV → JSON → HTML.
